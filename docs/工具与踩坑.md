## 2026-06-14 - LinX 添加流程真机 UI 测试

- **现象**：手点验证容易受当前页面状态影响；Xcode UI 测试启动后也会恢复上次停留页面，本机曾停在“已输注胰岛素”详情页，导致状态页 Settings 按钮不存在。
- **处理**：新增 `LoopUITests` scheme 和 `LoopCGMSetupUITests`，测试启动手机上已安装的 `com.libre.loopkit3.Loop`，先回到状态页，再进入 `Settings`、点击 `Add CGM`、选择 `MicroTech LinX`，最后确认 LinX 设置页和输入框出现。
- **前置条件**：手机已安装 `com.libre.loopkit3.Loop`；当前 App 不能已经配置 CGM；如果 `Add CGM` 不可见，测试会明确失败并提示先移除当前 CGM。
- **注意**：LinX 设置页的 SwiftUI `accessibilityIdentifier` 在当前 UI 层级里不稳定，测试同时使用页面文字和输入框占位文字兜底。
- **Watch 限制**：本机缺少 watchOS 26.5 时，绑定 App 构建的 UI test scheme 会在测试前失败；`LoopUITests` scheme 只构建测试包，正式 Watch 构建配置不改变。
- **验证**：`xcodebuild test -quiet -workspace LoopWorkspace.xcworkspace -scheme LoopUITests -configuration Debug -destination 'id=E30C92D5-FE26-5AE1-B5FB-C787E4401F4F' -allowProvisioningUpdates -allowProvisioningDeviceRegistration -disableAutomaticPackageResolution -only-testing:LoopUITests/LoopCGMSetupUITests/testMicroTechLinXSetupOpensFromSettings` 已通过；结果包 `Test-LoopUITests-2026.06.14_08-55-34-+0800.xcresult` 显示 `status=succeeded`、`testsCount=1`。

## 2026-06-14 - LinX 提示 Unable to Open CGM

- **现象**：在添加 CGM 时选择 `MicroTech LinX` 后，手机弹出 `Unable to Open CGM`。
- **原因**：LinX 插件工程引用了 Swift Package 生成的 `LoopKitUI_D1D06EAA165FD69_PackageProduct.framework`，但主 App 安装包里只有 `LoopKitUI.framework`，运行时加载 LinX 插件失败。
- **处理**：LinX 工程改为引用主 App 已包含的 `LoopKit.framework` 和 `LoopKitUI.framework`；同时将 LinX 目标最低 iOS 版本对齐到 15.1。
- **验证**：LinX 工程单独构建通过；主 App 构建通过；最终 `Loop.app` 内 LinX 插件已确认不再引用 `PackageProduct`；已覆盖安装并启动 `com.libre.loopkit3.Loop`。

## 2026-06-14 - Settings 中添加 CGM 页面被遮挡

- **现象**：在 `Settings` 中选择 `MicroTech LinX` 后，或已经添加 LinX 后再点现有 LinX 行，用户看不到后续设置页面，表现为点选后没有反应或卡住。
- **原因**：LinX 设置页本身是一个导航页面；添加 CGM 和打开现有 CGM 的入口从 `Settings` 返回后继续直接 `show`，可能进入不稳定的导航跳转路径。
- **处理**：新增 `presentAfterDismissingPresentedViewController`，添加 CGM、打开现有 CGM、打开 Pump 设置时先关闭当前页面或上层页面承载的弹窗，再弹出对应设置页面；CGM 打开失败时在手机上显示错误提示。
- **验证**：iPhone Debug 构建通过；安装包内确认存在 MicroTech 插件，插件显示名为 `MicroTech LinX`；已覆盖安装并启动 `com.libre.loopkit3.Loop`；`git diff --check` 已通过。
